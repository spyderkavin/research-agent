This project is a full-stack, multi-agent AI research platform that automates the entire process of gathering, synthesizing, and critically analyzing information on any topic. At its core, a research agent powered by the Gemini API autonomously breaks down a user's question into sub-questions, searches the web, reads full source pages, and compiles everything into a structured report with citations — all without any manual input beyond the initial question. Before research even begins, the user selects a political perspective on a five-point spectrum from Left to Right, and the agent dynamically adjusts its sourcing strategy, language, and framing to match that lens. A secondary controversy detection agent then evaluates the topic and, if deemed even slightly sensitive, triggers a bias analysis agent that reads each source individually and classifies its political lean with a written explanation. The entire system is accessible both through a terminal interface for development and testing, and through a custom-built web frontend served by a FastAPI backend, with a GitHub Actions CI pipeline that automatically runs the test suite on every push to ensure nothing breaks.

---

- **Agentic Architecture** — the system uses multi-step tool calling where the AI autonomously decides what to search, what pages to read, and when it has enough information to write a final report — no hardcoded steps
- **Perspective-Based Framing** — users select from five political lenses (Left, Center-Left, Neutral, Center-Right, Right) and the agent adjusts its sourcing and language accordingly before research begins
- **Automated Controversy Detection** — a dedicated agent checks every topic for sensitivity and conditionally triggers the bias pipeline, meaning neutral topics like "how does gravity work" skip analysis entirely
- **Per-Source Bias Scoring** — each source is read in full and scored individually on the Left-to-Right spectrum with a one-sentence explanation of why, rather than applying a blanket label to the whole report
- **Dual Interface** — runs in the terminal for CI/testing purposes and in the browser via a dark-themed HTML/CSS/JS frontend connected to a FastAPI backend
- **CI Pipeline** — GitHub Actions automatically runs 20+ unit tests with mocked API calls on every push, enforcing 70% minimum code coverage so the codebase stays reliable as it grows
